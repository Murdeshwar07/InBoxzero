The "Logic" ‚Äî Confidence Thresholds

This is the superpower of an AI PM. The model is never 100% sure. It outputs a probability score (0.0 to 1.0). You, the PM, must decide the cut-off line.

The Probability Problem

Every ML classification model outputs a confidence score‚Äîa number between 0 and 1 representing how certain it is about its prediction. A score of 0.85 means "I'm 85% confident this is a Refund Request."

The model will never say "I'm 100% sure." That's not how probability works. So you, the PM, have to decide: at what confidence level do we trust the machine to act autonomously?

Credit Card Fraud Detection

99% confidence it's fraud? Block the card immediately. No human needed.
60% confidence? Send a text asking "Was this you?" Human verification.
10% confidence? Do nothing. Let the transaction through.

üí° Why Flag for Review?

At 85% confidence, there's still a 15% chance this is a real customer email. In e-commerce, deleting a legitimate customer complaint could mean losing a customer forever‚Äîand a potential PR nightmare if they post about it on social media. The cost of a false positive (deleting real mail) far exceeds the cost of a false negative (reviewing spam). For SPAM specifically, we set the auto-delete threshold at 95%+.

Threshold Philosophy by Category

Not all misclassifications are equal. The threshold should vary based on the cost of error:

Order Status (85% threshold): Low-risk. If we route a refund request to the tracking team, they'll just forward it. Minor friction.

Returns & Refunds (90% threshold): Medium-risk. Refund requests need agents with approval authority. Misrouting wastes time.

SPAM Detection (95% threshold): High-risk. False positives = deleted customer emails. We need near-certainty before auto-deleting.

VIP/Escalation (80% threshold): Inverse risk. We'd rather over-escalate than miss an angry whale customer. Lower threshold = more false positives, but that's acceptable.
